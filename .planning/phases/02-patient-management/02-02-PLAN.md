Add HTMX page-serving endpoints to the existing patients.py router (created in Plan 02-01). These endpoints return HTML fragments or full pages, not JSON. They are distinct from the JSON API endpoints.

Add these new endpoints after the existing CRUD endpoints:

1. **GET /patients/page** (returns HTML):
   - require_permission("patient", "read")
   - Renders the split-view patient list page (app/templates/patients/list.html)
   - Returns a Jinja2 HTMLResponse
   - This is the main entry point for the patient management UI

2. **GET /patients/register/page** (returns HTML):
   - require_permission("patient", "create")
   - Renders the 4-step registration wizard (app/templates/patients/register.html)
   - Returns HTMLResponse

3. **GET /patients/{patient_id}/page** (returns HTML):
   - require_permission("patient", "read")
   - Fetches patient with emergency_contacts and next_of_kin (use selectinload)
   - Renders patient profile (app/templates/patients/profile.html)
   - Returns HTMLResponse
   - 404 if not found

4. **GET /patients/search** (returns HTML fragment):
   - require_permission("patient", "read")
   - Query param: q (search term, min 4 chars)
   - Calls PatientService.search_patients() or performs direct search
   - Returns HTML fragment with patient cards (uses patient_card.html component)
   - D-68: Log every search via AuditService

5. **GET /patients/list** (returns HTML fragment):
   - require_permission("patient", "read")
   - Query params: page, page_size
   - Returns HTML fragment with patient list table rows

6. **GET /patients/{patient_id}/detail** (returns HTML fragment):
   - require_permission("patient", "read")
   - Returns HTML fragment for the right panel of split view
   - Includes patient demographics, emergency contacts, next of kin

Required imports to add:
```python
from fastapi.responses import HTMLResponse
from jinja2 import Environment, FileSystemLoader
from pathlib import Path
```

Template rendering approach: Use Jinja2 Environment with FileSystemLoader pointed at app/templates/. Alternatively, if the project uses FastAPI's Jinja2Templates integration, use that. Check how admin templates are served (likely via a template endpoint in the admin router).

Search implementation for endpoint 4: Use SQLAlchemy with ilike for basic search (pg_trgm similarity is PostgreSQL-specific and won't work in SQLite tests). Build search with OR conditions on first_name, last_name, mrn, phone using ilike. Order by relevance (exact match first, then partial).

D-68 audit logging: Every search call must invoke AuditService.log_event with action="patient_search", resource="patient", metadata={"query": q, "results_count": count}.
