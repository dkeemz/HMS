## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → API | Patient PII (name, DOB, phone, address, NIN) crosses from browser to FastAPI endpoints |
| API → Database | Patient data persisted to PostgreSQL; MRN generated via DB sequences |
| API → Audit | Every patient mutation and search must be logged to AuditLog (HIPAA) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | Patient registration input | mitigate | Pydantic PatientCreate validates all fields; Nigerian phone regex enforced; gender constrained to Male/Female |
| T-02-02 | Information Disclosure | Patient search results | mitigate | require_permission("patient", "read") on all GET endpoints; D-68 search audit logging |
| T-02-03 | Information Disclosure | MRN enumeration | accept | MRN is not a secret (staff use it); search is audit-logged |
| T-02-04 | Tampering | SQL injection via search | mitigate | SQLAlchemy ORM parameterized queries; no raw SQL with user input |
| T-02-05 | Elevation of Privilege | Unauthorized patient access | mitigate | require_permission() on every endpoint; role-based via RBACService |
| T-02-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed (all deps already in project) |
